IDFA 比对

钱咖需要通过本接口来获取某一个 IDFA 是否下载过广告主的应用。

钱咖有着数百万的用户，对于装机量大的应用，很有可能有相当一部分的钱咖用户下载过这些应用。将广告推给已安装过广告主，不仅会影响推广效果，更会使得广告主和钱咖在推广结束后由于数据核对问题影响后续合作，错过最佳推广时机。另外，很多渠道也存在重复用户的情况，所以钱咖建议广告主在推广前对接好 IDFA 比对接口，达到双赢的目的。

技术实现

IDFA 比对接口通过 http 或 https 协议，由钱咖向广告主发起。广告主需要搭建 Web 服务并告知钱咖该服务的 URL；当接收到来自钱咖发起的请求时，返回比对结果。

例如：

						http://example.com/idfa-query

钱咖通过 POST 方法（application/x-www-form-urlencoded）提交以下参数至广告主提供的 URL。参数说明如下，

						appid：广告主应用在 App Store 的编号。
						idfa: IDFA 列表，多个以逗号","分隔。钱咖一次查询最多发送 1,000 个 IDFA。例如：
						idfa=5A58EF1E-EEF2-478D-94EE-709B98407589,A0A82816-3383-437B-A535-F910162A7098

返回结果为 json 格式，如 IDFA 已存在标识为 1，不存在标识为 0。例如：

						{
						  "5A58EF1E-EEF2-478D-94EE-709B98407589": 1,
						  "A0A82816-3383-437B-A535-F910162A7098": 0
						}
